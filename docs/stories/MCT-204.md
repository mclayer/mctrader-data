# MCT-204: compactor _tick stall 해소 + historical L1 회수 경로 신설

- **Issue**: [mclayer/mctrader-data#184](https://github.com/mclayer/mctrader-data/issues/184)
- **Spec**: [docs/superpowers/specs/2026-05-19-compactor-tick-stall-historical-l1-reclaim.md](../superpowers/specs/2026-05-19-compactor-tick-stall-historical-l1-reclaim.md)
- **ADR**: mctrader-hub:docs/adr/ADR-027-cold-tier-object-storage-nas-minio.md (§D5/§D7 amendments, TBD) + mctrader-hub:docs/adr/ADR-029 (D1=B amendment, TBD)

## §1 사용자 요구사항

> 1. "이전에 compact 완료된 데이터는 삭제하라 했는데 여전히 해결되지 않아 계속 로컬 디스크가 낭비되고 있다."
> 2. "함께 L3로 데이터도 잘 쌓이고 있지 않다."

## §2 근본 원인 — single origin (RequirementsPLAgent + DomainAgent 통합)

두 사용자 호소는 **단일 origin**: forward `_run_l2` worker thread 가 NAS GET 안에서 stall → 같은 `_tick` 안 후속 step (`_run_l3`, `_cycle_count++`, `scan_and_cleanup_legacy`) 영원히 진입 못함.

### §2.1 sequential 차단 chain (runner.py)

| step | 위치 | 동작 | 차단 영향 |
|---|---|---|---|
| 1 | `runner.py:107-112` | `for sealed in sealed_list: self._l1.compact_segment(sealed)` | 정상 (L1 dispatch=1914) |
| 2 | `runner.py:114-116` | `await run_in_executor(None, self._run_l2)` | **blocking await** — worker stall 시 main thread 영원 대기 |
| 3 | `runner.py:118-120` | `await run_in_executor(None, self._run_l3)` | **2 차단 시 도달 0** (L3 dispatch=0, 사용자 호소 #2) |
| 4 | `runner.py:131` | `self._cycle_count += 1` | 2 차단 시 영원히 0 |
| 5 | `runner.py:132-145` | `if cycle_count % LEGACY_CLEANUP_EVERY_N_CYCLES == 0: scan_and_cleanup_legacy(...)` | 2 차단 시 호출 0 (사용자 호소 #1) |

### §2.2 worker stall 원인 — rglob outer iter 폭주

| 위치 | 코드 | 측정 결과 |
|---|---|---|
| `runner.py:158` | `(self._root / "market").rglob("*/tier=L1/**/part-*.parquet")` | **historical 16,918 file iter** (5/13~17 130GB 누적) |
| `runner.py:160-173` | 발견 partition × `[today, yesterday]` × 24 hour seen 중복 제거 | 400 unique (e,s,c) tuple × 48 dispatch = **19,200 worker NAS GET** |
| `runner.py:211` | `(self._root / "market").rglob("*/tier=L2/**/part-*.parquet")` | L3 동형 |

→ rglob outer iter 가 today/yesterday window 와 무관한 historical partition 까지 매번 scan. 평균 6 sec/dispatch × 19,200 ≈ 32h/cycle. py-spy 박제 (Thread 133 = `get_streaming` → `_compact_hour_nas` → `_run_l2_for_parquet`, idle 2h+).

### §2.3 historical L1 회수 경로 부재

| 측정 | 결과 |
|---|---|
| operator one-shot `scan_and_cleanup_legacy` (31 iter, batch 500) | L2 10.8GB 회수 / **L1 0 byte 회수** (iter 10+ 매 batch `cleaned=0 preserved=500`) |

`scan_and_cleanup_legacy` 의 회수 gate = L1/L2 양쪽 NAS HEAD 4-tuple verify pass. `promote_historical` 는 L1 → L2 변환 + L2 NAS PUT 만 수행 (`runner.py:471-473` 주석 `source_to_delete=None`, MCT-202 D-3 cascade 의도 보존). L1 NAS 객체 부재 → 4-HEAD verify fail → preserved.

→ **historical L1 130GB 회수 경로가 본질적으로 부재** (forward eager cascade = today/yesterday only, promote-historical = L2 만 PUT, sweep = NAS HEAD 의존). 신규 회수 경로 신설 필요.

### §2.4 ADR-027 §D5 incident pattern 정합

- ADR-027 §D5: "NAS PUT 4xx fail-fast — silent fallback 차단" (2026-05-18 amendment, mctrader-hub SSOT)
- 본 origin = **silent stall** (raise 없음, retry_queue 0 흡수, 사용자 인지 없음) — §D5 amendment 의도 (silent failure 차단) 확장 필요. ADR-027 §D5 amendment 추가 (cooperative scheduling head-of-line blocking pattern 박제).

## §6 요구사항 명세 — Acceptance Criteria (RequirementsPLAgent owner)

5+ AC, Given/When/Then 형식. Layer 1+2+3 통합 cross-cut.

### AC-1: forward `_tick` 차단 해소 (Layer 1+2 통합)

- **Given**: forward `_run_l2` worker thread 가 NAS GET 안에서 stall (현재 stall 시간 2h+ 박제)
- **When**: `_tick` next iteration 진입
- **Then**: `scan_and_cleanup_legacy` 호출이 worker stall 과 무관하게 **자기 cadence (≤ 12 × `SCAN_INTERVAL_SECONDS`, default ≤ 5분)** 으로 진행
- **측정** (**FIX 1/3 P1 #11 Codex 4** — restart edge case 면제 wording 추가): `mctrader_compactor_cleanup_cycle_delay_seconds` Gauge ≤ **300s steady state** (Prometheus). **container restart 직후 첫 1 cycle 면제** (≤ 360s = `SCAN_INTERVAL_SECONDS 30s × LEGACY_CLEANUP_EVERY_N_CYCLES 12`). Prometheus alert rule = `(cleanup_cycle_delay > 300) and (up{job="compactor"} == 1 for 1m)` (restart 직후 1분 면제).
- **현재 baseline**: ∞ (영원히 도달 안 함)

### AC-2: forward `_run_l2` rglob 범위 축소 (Layer 1)

- **Given**: `<root>/market/<channel>/` 안 L1 파일 16,918 개 (5/13~17 historical 130 GB 포함)
- **When**: forward `_run_l2` 1 invocation 수행
- **Then**: rglob 호출이 iter 하는 file 수 = **forward window (`today-1`, `today`) 안 partition 의 file 수만**, historical (date < today-1) iter 0
- **측정**: integration test `test_compactor_forward_rglob_scope.py` 가 historical partition 생성 후 `_run_l2` 호출 시 historical partition file open count = 0 박제
- **수치 목표**: forward partition file 수 × 1.2 이하 (안전 margin)
- **현재 baseline**: 16,918 file iter (forward + historical 무차별)

### AC-3: historical L1 회수 경로 작동 (Layer 3)
